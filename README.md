# Prefect Learning Content

Hands-on learning materials for Prefect training, workshops, and self-guided labs.

This repository is where we build and maintain the learning materials published at [prefect.io/learn](https://prefect.io/learn). It gives us a shared place to iterate on content in public, incorporate feedback, track changes over time, and use agentic workflows and skills to produce content faster without sacrificing quality.

## Why This Repo Exists

Prefect is easier to learn when people can practice it in realistic labs, not just read about it.

We use this repo to:

- Build course content in public and improve it with feedback
- Keep learning materials version-controlled and easy to review
- Build and update content with agentic workflows and skills
- Make it easier for external contributors to improve the learning experience

## What’s Inside

We use [Instruqt](https://docs.instruqt.com/) to run these courses, and the repository follows Instruqt's track-and-challenge structure.

Each course lives in `courses/<course-slug>/` and usually includes:

- `track.yml` for metadata and course configuration
- One or more challenge directories with `assignment.md` instructions
- Supporting assets such as images, logos, and diagrams

## Who This Is For

This repo is for:

- Content developers creating new courses and challenges
- Maintainers updating existing learning materials
- Instructors and workshop hosts adapting course content for live delivery
- External contributors improving the clarity, accuracy, and usefulness of the material

It is especially useful for teams and contributors who want a structured, version-controlled workflow for building and maintaining hands-on technical learning content.

## How to Contribute

You do not need to work at Prefect to help improve these learning materials.

Good contributions include:

- Fixing typos, unclear wording, or broken links
- Improving challenge instructions so they are easier to follow
- Adding examples that reflect real-world usage
- Updating screenshots, diagrams, or assets
- Reporting content that feels outdated, confusing, or incomplete
- Suggesting new topics that would help people learn faster

If you want to contribute:

1. Open an issue or discussion for the change you want to make.
2. Make a small, focused pull request.
3. Describe the learning problem your change solves and who it helps.

When possible, keep edits focused on making the material clearer, more accurate, and easier to maintain.
